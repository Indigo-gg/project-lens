# Project-Lens V5 重构计划

> **基于**: 二哥最新指导 `docs/erge-plus2.md`
> **核心原则**: Lens维护的是"项目知识"，Agent维护的是"职业知识"
> **设计原则**: Lens回答 What Exists（代码里有什么事实），Agent回答 What It Means（这些事实意味着什么）

---

## 〇、V4 → V5 核心变化

| 变化 | V4 | V5 | 原因 |
|:---|:---|:---|:---|
| 工具数量 | 10 个 | **5 个** | 复杂度过高，Agent不需要 |
| Capability模块 | extract_capabilities + match_capabilities | **删除** | Agent能自己总结，Redis≠Caching是Interpretation |
| JD Parser | NER分类→Requirements | **→ Search Query** | Claude已经能读懂JD，不需要Lens做分类 |
| Capability Graph | capabilities + capability_facts表 | **删除** | 违反"What Exists"原则 |
| export_context | Repository Memory | **→ export_snapshot** | 改名Project Snapshot |
| Evidence排序 | 无 | **Evidence Ranking** | Top-N返回价值巨大 |
| search_evidence | 简单FTS | **Universal Query** | 所有查询统一到search_evidence的参数 |
| decision_trace | 独立工具 | **→ search_evidence(relation='decision')** | 统一到search_evidence |

## 一、五个核心工具

### Tool 1: analyze_project()
扫描项目，建立索引（Facts + Edges + Evidence）

```typescript
// Input
{ project_path?: string, depth?: number, force_reindex?: boolean }
// Output
{ project: { name, languages, dependencies, total_files, loc },
  stats: { facts, edges, evidences, decision_traces },
  modules: Array<{ path, facts, entry_points }>, indexed_at: string }
```

**实现**: 复用 `file-scanner.ts` + `store.ts`，新增 `ast-parser.ts`、`fact-builder.ts`、`git-signal.ts`

### Tool 2: search_evidence()
万能搜索入口，所有查询统一到这一个工具

```typescript
// Input (所有参数可选，组合使用)
{ query?: string, category?: string, evidence_type?: string, fact_type?: string,
  relation?: string, requirement?: string, sort_by?: "score"|"recent"|"author",
  limit?: number, cursor?: string }
// Output
{ results: Array<{ fact, evidence, score, decision_trace? }>,
  next_cursor?: string, total_count: number }
```

**Evidence Ranking 公式**:
```
Score = 0.25×Benchmark + 0.20×Tests + 0.20×GitEvolution + 0.10×LOC + 0.15×Recency + 0.10×Complexity
```

**Requirement Search**: `search_evidence({ requirement: 'Redis' })` → 自动扩展为 `["redis", "ioredis", "cache", "pub/sub", "session"]`

### Tool 3: verify_statement()
验证一句话是否有代码证据支撑

```typescript
// Input: { statement: string, context_scope?: string[] }
// Output: { verdict: "SUPPORTED"|"PARTIALLY_SUPPORTED"|"NOT_SUPPORTED",
//           confidence: number, evidence: Array<...>, unsupported_parts: string[] }
```

**实现**: 关键词提取 → search_evidence → 置信度计算 → 判定

### Tool 4: export_snapshot()
导出项目知识包（Project Snapshot）

```typescript
// Input: { format?: "json"|"compact", include_git_history?: boolean, max_tokens?: number }
// Output: { project, key_facts, recent_decisions, evidence_stats, top_evidence }
```

**压缩策略**: 保留摘要+Top20+最近30天Decision Trace，丢弃完整代码+完整历史，目标<50K tokens

### Tool 5: render_resume()
将resume JSON渲染为PDF

```typescript
// Input: { resume_json: string, template?: string }
// Output: { pdf_path: string }
```

## 三、数据库Schema变更

### 删除的表
```sql
DROP TABLE IF EXISTS capabilities;
DROP TABLE IF EXISTS capability_facts;
DROP TABLE IF EXISTS jd_requirements;
```

### 新增的表
```sql
-- FTS5 全文搜索
CREATE VIRTUAL TABLE nodes_fts USING fts5(name, qualified_name, file_path, properties, content=nodes, content_rowid=id);
CREATE VIRTUAL TABLE evidence_fts USING fts5(description, commit_hash, author, content=evidence, content_rowid=id);
CREATE VIRTUAL TABLE decision_traces_fts USING fts5(ast_change, description, content=decision_traces, content_rowid=id);

-- Requirement Synonyms
CREATE TABLE requirement_synonyms (
  requirement TEXT PRIMARY KEY,
  search_terms TEXT NOT NULL,  -- JSON array
  category TEXT
);
```

### 修改的表
- `nodes`: 新增 `content_hash`, `loc` 字段
- `evidence`: 新增 `evidence_score` 字段
- `projects`: 新增 `languages`, `dependencies`, `total_files`, `loc` 字段

## 六、数据流场景

### 场景1: Agent搜索项目亮点
```
search_evidence({ category: "performance" })
→ requirement-expander扩展 → FTS5搜索 → ranking评分 → 返回Top 20
```

### 场景2: Agent验证简历陈述
```
verify_statement({ statement: "优化了Redis缓存性能" })
→ 关键词提取 → search_evidence → 置信度计算 → 返回verdict
```

### 场景3: Agent搜索JD相关证据
```
search_evidence({ requirement: "Redis" })
→ 自动扩展["redis","ioredis","cache","pub/sub"] → FTS5搜索 → ranking → Top 10
```

### 场景4: Agent获取项目全貌
```
export_snapshot({ format: "compact" })
→ 查询nodes+evidence+decisions → 压缩 → Token估算 → <50K tokens
```

### 场景5: Agent追溯决策历史
```
search_evidence({ relation: "decision", query: "Redis" })
→ 搜索decision_traces → 按时间排序 → 返回时间线
```
