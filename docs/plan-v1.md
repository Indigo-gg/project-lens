这是一份为您量身定制的“简历定制与项目级AI模拟面试闭环系统”项目计划书。该计划书通过整合当前高度成熟的开源项目，以最小的开发成本实现您的核心需求。

### 项目名称：Project-Lens（AI 驱动的简历组装与深度模拟面试平台）

#### 一、 项目核心目标

构建一个一站式求职准备平台，实现两大核心闭环：

1. **模块化简历构建**：用户可以维护一个“项目池”，根据不同岗位的需求，勾选并动态组装、优化出最匹配的定制化简历。
2. **基于项目的深度面经**：系统读取组装好的简历数据，针对用户勾选的特定项目，自动生成情景题、技术八股文，并在用户作答后进行评分和提供基于该项目技术栈的标准参考答案。

---

#### 二、 核心架构与开源项目参考价值

为了避免从零开发，本系统将采用前后端分离架构，直接复用以下优秀的开源项目源码与逻辑：

**1. 简历数据管理与前端底座**

* **参考项目**：**ResumeLM** (作者: olyaiy)


* **参考价值**：该项目提供了工业级的 Next.js 15 + React 19 前端架构，以及基于 PostgreSQL 和 Supabase 的后端数据结构。其核心价值在于已经实现了“基线简历（Base Resumes）”与“定向变体（Tailored Versions）”的分层逻辑。您可以直接复用其数据库设计，将您的“成熟项目”存入 `Profiles` 表的 JSON 字段中，通过其界面实现项目的勾选、组装和导出定制版简历。



**2. 模拟面试工作流与智能体编排**

* **参考项目**：**CareerLens_AI** (作者: Ladanividhi)


* **参考价值**：这是一个基于 LangGraph 和 FastAPI 构建的多智能体系统。它已经完整实现了“解析简历 -> 生成特定类型面试题（技术/HR/场景） -> 评估用户回答（打分、指出错误、提供建议）”的 API 接口。您可以将其作为核心后端，只需要在此基础上修改其大模型系统提示词（Prompt），强制要求它在出题和生成标准答案时，必须以传入的“项目 JSON 详情”为唯一上下文来源。



**3. 面试交互界面与评分面板**

* **参考项目**：**Ai-mock-Interview** (作者: modamaan)


* **参考价值**：该平台提供了一个非常完善的模拟面试交互体验界面，支持根据经验等级和岗位生成问题，并包含详细的反馈洞察和整体评分系统。您可以提取其 UI 组件（使用 Tailwind CSS 构建），无缝拼接到 ResumeLM 的工作台中，为用户提供一个沉浸式的在线答题和查看标准答案的界面。

#### 三、 核心功能实现路径（二次开发指南）

**第一步：构建“项目池”与简历动态组装（复用 ResumeLM）**

1. 部署 Supabase 数据库，复用 ResumeLM 的 `Profiles` 表结构，但在此基础上新增一个 `Project_Pool` 模块。
2. 在前端界面中，用户可以预先录入 5-6 个成熟项目的详细信息（包括业务背景、技术栈、核心难点、个人贡献）。
3. 投递特定岗位时，用户在界面上勾选 2 个最相关的项目，系统利用内置的大模型能力将这些项目润色并输出为一份结构化的 JSON 简历数据，同时生成对应的 PDF。

**第二步：改造面试引擎（注入 CareerLens_AI 逻辑）**

1. 将第一步生成的 JSON 简历数据（仅包含被选中的项目）传递给基于 CareerLens_AI 修改的 FastAPI 后端。
2. **重写出题 Prompt**：要求大模型根据传递的 JSON 节点出题。例如：“针对用户在简历中提到的『使用 Redis 解决高并发缓存击穿』项目，生成一道场景应对题和一道底层原理（八股文）题。”
3. **预生成标准答案**：在用户答题前，系统后台大模型必须首先利用该项目的技术栈生成一份“标准参考答案（Rubric）”并暂时隐藏。

**第三步：答题与智能评估（融合 Ai-mock-Interview UI）**

1. 在前端展示题目，用户可以通过文本或语音输入答案。
2. 提交后，后端对比用户的输入与第二步生成的“标准参考答案”。
3. 最终输出一份详细的报告：包含当前回答得分（1-10分）、优点、逻辑漏洞、以及严格基于简历项目背景的“完美参考答案”。

#### 四、 阶段实施计划

* **Phase 1：基础设施搭建 (第 1-2 周)**
* 克隆并本地部署 ResumeLM 和 Supabase 环境，跑通基础的登录与简历数据 CRUD 流程。
* 修改其数据库表结构，增加用户“项目池”的概念。


* **Phase 2：面试后端引擎改造 (第 3-4 周)**
* 提取 CareerLens_AI 的核心 Python 后端代码，剥离不需要的模块，仅保留 `start-interview` 和 `evaluate` 相关的端点。


* 调试 Prompt 工程，确保 AI 严格基于 JSON 项目数据出题，杜绝生成与简历无关的通用面试题。


* **Phase 3：前端整合与联调 (第 5-6 周)**
* 将 Ai-mock-Interview 的交互式答题界面组件整合到 ResumeLM 的 Next.js 前端体系中。
* 打通数据流：从勾选项目 -> 生成简历 -> 触发面试 -> 结果评分的完整闭环。



通过这种“搭积木”的方式，您可以省去编写底层数据库校验、PDF排版引擎和基础 LLM 接口调用的时间，将核心精力全部集中在打磨“基于项目的定制化 Prompt”上。
